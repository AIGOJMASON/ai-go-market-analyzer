# CHILD CORE OUTPUT POLICY (NORTHSTAR ALIGNED)

## PURPOSE

Defines how outputs behave.

---

## OUTPUT TYPES

Child cores may emit:

- advisory outputs
- structured results
- bounded artifacts

---

## OUTPUT REQUIREMENTS

All outputs MUST:

- map to curated input
- remain within domain scope
- expose authority limits

---

## PERSISTENCE LAW

If output is stored:

Persistence = Mutation  
→ must pass full governance chain

---

## FORBIDDEN

Child cores may NOT:

- write directly to state
- create hidden memory
- persist uncontrolled data

---

## SUMMARY

Outputs are:

→ CONTROLLED  
→ TRACEABLE  
→ NON-AUTHORITATIVE
