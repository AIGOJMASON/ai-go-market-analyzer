# DISPATCH TO INGRESS

## PURPOSE

This document defines the lawful transition from dispatch posture into child-core ingress.

Dispatch does not send raw source material to a child core.

Dispatch may only route bounded, curated, adapter-shaped context into ingress.

## REQUIRED FLOW

All child-core dispatch-to-ingress movement must preserve:

source or provider signal  
→ RESEARCH_CORE  
→ engine curation  
→ adapter  
→ child core  

The adapter is the final shaping layer before child-core ingress.

## DISPATCH ROLE

Dispatch may:

- identify the target child core
- attach routing metadata
- confirm adapter output exists
- hand bounded context to ingress
- preserve advisory status
- preserve execution_allowed = false

Dispatch may not:

- fetch providers
- bypass RESEARCH_CORE
- bypass engine curation
- bypass adapter shaping
- mutate workflow
- create execution permission
- create recommendation authority

## INGRESS ACCEPTANCE RULE

Ingress may accept dispatch only when the packet includes:

- RESEARCH_CORE lineage
- engine curation marker
- adapter identity
- child-core target
- bounded operating scope
- authority boundary
- execution_allowed = false

## MUTATION LAW

Persistence = Mutation.

Dispatch-to-ingress is not mutation by default.

If dispatch evidence, ingress record, receipt, or routing trace is persisted, that persistence must be explicit, classified, traceable, and governed.

All mutation must pass:

State Authority  
→ Watcher  
→ Canon  
→ Request Governor  
→ Execution Gate  
→ Cross-Core Integrity  

## RECEIPT LAW

A receipt is a persisted artifact.

A receipt may document dispatch-to-ingress movement.

A receipt may not authorize execution.

A receipt may not mutate workflow truth.

A receipt may not become child-core authority.

## EXECUTION LAW

execution_allowed = false by default.

Dispatch-to-ingress cannot trigger execution.

It only moves lawful bounded context into child-core ingress.

## SUMMARY

Dispatch-to-ingress is a routing boundary, not an authority boundary.

It preserves lawful upstream flow and keeps the child core limited to curated adapter-shaped context.
