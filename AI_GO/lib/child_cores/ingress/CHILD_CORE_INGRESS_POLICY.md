# CHILD CORE INGRESS POLICY

## PURPOSE

This document defines lawful ingress into any AI_GO child core.

Child-core ingress is not raw intake.

Child-core ingress is the controlled point where bounded, curated, adapter-shaped context becomes available to a child core.

## CHILD-CORE FLOW LAW

All child-core input must follow:

source or provider signal  
→ RESEARCH_CORE  
→ engine curation  
→ adapter  
→ child core  

No child core may receive raw provider material as authority.

Provider retrieval is reserved for RESEARCH_CORE only. Child-core scope begins after engine curation and adapter shaping.

No child core may interpret external source material before RESEARCH_CORE and engine curation.

## INGRESS AUTHORITY

Ingress may:

- accept adapter-shaped context
- validate child-core target identity
- confirm schema shape
- confirm authority metadata
- prepare bounded runtime context

Ingress may not:

- fetch external providers
- mutate workflow
- mutate project truth
- write memory
- write outcome feedback
- create execution permission
- create recommendation authority

## REQUIRED INGRESS METADATA

A lawful ingress packet should expose:

- source authority
- RESEARCH_CORE lineage
- engine curation marker
- adapter identity
- child core target
- scope boundary
- advisory status
- execution_allowed = false
- mutation_allowed = false unless governed

## MUTATION LAW

Persistence = Mutation.

Ingress does not mutate by default.

If ingress evidence, receipt, outcome, or runtime context is persisted, that persistence must be explicit, classified, traceable, and governed.

All mutation must pass:

State Authority  
→ Watcher  
→ Canon  
→ Request Governor  
→ Execution Gate  
→ Cross-Core Integrity  

## RECEIPT LAW

A receipt is a persisted artifact.

Therefore:

Persistence = Mutation.

Any ingress receipt must be classified before persistence.

A receipt may record ingress posture, but it may not create authority.

## OUTCOME LAW

Ingress may not create outcome truth.

Any outcome reference is advisory only.

Outcome persistence is classified mutation and cannot alter workflow or decision authority.

## EXECUTION LAW

execution_allowed = false by default.

Ingress cannot trigger execution.

Ingress cannot authorize child-core action beyond bounded context availability.

## SUMMARY

Child-core ingress is a controlled handoff boundary.

It receives only curated, adapter-shaped context from lawful upstream flow and cannot mutate, execute, or create authority.
