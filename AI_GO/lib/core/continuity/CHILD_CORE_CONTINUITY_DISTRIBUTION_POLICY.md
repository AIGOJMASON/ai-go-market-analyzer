# CONTINUITY DISTRIBUTION → CONSUMPTION

## Purpose

Defines how continuity data moves from distribution layer to child-core consumption.

---

## Upstream Source Law

Continuity originates from governed flow:

source or provider signal  
→ RESEARCH_CORE  
→ engine interpretation  
→ adapter shaping  
→ PM integration  
→ continuity distribution  

---

## Distribution Rules

Continuity distribution must:

- preserve source traceability
- maintain classification (advisory vs mutation candidate)
- avoid expansion or reinterpretation

---

## Consumption Rules

Child cores:

- consume continuity as advisory input
- do not treat continuity as authority for mutation

---

## Mutation Handling

Any mention of:

- receipt
- persistence
- memory
- outcome

is considered **mutation**

---

## Mutation Enforcement

All mutation must pass:

State Authority  
→ Watcher  
→ Canon  
→ Request Governor  
→ Execution Gate  
→ Cross-core enforcement  

No exceptions.

---

## Prohibited Behavior

- distribution that implies execution
- mutation implied without classification
- continuity used as direct action trigger

---

## Summary

Distribution delivers context.

Consumption interprets context.

Mutation requires governance.
