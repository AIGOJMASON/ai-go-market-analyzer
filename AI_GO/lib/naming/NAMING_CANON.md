# NAMING CANON

## Purpose

Defines naming standards across the system.

---

## Governance Alignment

Naming must reflect:

- system structure
- authority boundaries
- ingestion and mutation laws

---

## Mutation Language Rule

Any naming involving:

- receipt
- memory
- persistence
- outcome

must be treated as **mutation-related terminology**

---

## Mutation Enforcement

All mutation-related constructs must align with:

State Authority  
→ Watcher  
→ Canon  
→ Request Governor  
→ Execution Gate  
→ Cross-core enforcement  

---

## Prohibited Behavior

- naming that implies uncontrolled mutation
- naming that hides governance responsibility

---

## Summary

Names must reflect truth.

Truth includes governance.
