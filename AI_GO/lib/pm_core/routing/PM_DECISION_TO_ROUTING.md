# PM DECISION TO ROUTING

## PURPOSE

Defines lawful transition from PM decision posture into routing.

PM decision-to-routing is bounded selection movement.

It is not execution.

It is not mutation by default.

It does not create authority by itself.

## REQUIRED FLOW

Any PM decision-to-routing path involving child-core context must preserve:

source or provider signal  
→ RESEARCH_CORE  
→ engine curation  
→ adapter  
→ PM interpretation  
→ PM decision posture  
→ PM routing  
→ child-core ingress when dispatched  

Provider retrieval is reserved for RESEARCH_CORE only.

Child-core routing context must be curated, adapter-shaped, bounded, and advisory unless governed mutation is explicitly approved.

## DECISION POSTURE MAY

- identify a lawful routing direction
- preserve lineage metadata
- preserve authority boundaries
- preserve execution_allowed = false
- prepare bounded routing context

## DECISION POSTURE MAY NOT

- execute actions
- mutate workflow directly
- mutate project truth directly
- write memory outside governance
- write outcome feedback outside governance
- create child-core authority
- create final execution authority

## ROUTING MAY

- select lawful target boundary
- preserve advisory status
- preserve mutation_allowed = false unless governed
- prepare dispatch posture

## ROUTING MAY NOT

- execute actions
- mutate workflow directly
- mutate project truth directly
- override governance
- authorize downstream action by itself

## MUTATION LAW

Persistence = Mutation.

If decision evidence, routing evidence, receipt, routing trace, dashboard packet, memory record, outcome reference, latest snapshot, or routing artifact is persisted, it must be explicit, classified, traceable, and governed.

All mutation must pass:

State Authority  
→ Watcher  
→ Canon  
→ Request Governor  
→ Execution Gate  
→ Cross-Core Integrity  

## RECEIPT LAW

A receipt is a persisted artifact.

A receipt may document governed decision-to-routing posture.

A receipt may not authorize execution.

A receipt may not mutate workflow truth by itself.

## EXECUTION LAW

execution_allowed = false by default.

PM decision-to-routing cannot trigger execution.

## SUMMARY

PM decision-to-routing prepares bounded advisory routing only.

It cannot mutate, execute, retrieve providers, or create authority.
