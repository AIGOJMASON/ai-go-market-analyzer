# CHILD CORE CONTINUITY POLICY

## PURPOSE

This document defines lawful continuity handling for AI_GO child-core context.

Continuity is advisory context.

Continuity is not execution.

Continuity is not authority creation.

Continuity is not persistence unless explicitly governed.

## CHILD-CORE FLOW LAW

All child-core continuity context must preserve lawful upstream lineage:

source or provider signal  
→ RESEARCH_CORE  
→ engine curation  
→ adapter  
→ child-core ingress  
→ child-core runtime  
→ child-core output  
→ child-core review  
→ child-core watcher  
→ continuity  

Provider retrieval is reserved for RESEARCH_CORE only.

Continuity may only consume bounded context created through curated, adapter-shaped upstream flow.

## CONTINUITY ROLE

Continuity may:

- preserve governed context
- expose historical posture
- support review
- support operator understanding
- support advisory system memory
- support bounded child-core context recovery

Continuity may not:

- execute actions
- mutate workflow directly
- mutate project truth directly
- create recommendation authority
- override Request Governor
- override Watcher
- override Canon
- override Execution Gate
- override State Authority
- override Cross-Core Integrity

## MUTATION LAW

Persistence = Mutation.

Continuity read is not mutation.

Continuity persistence is mutation.

If continuity evidence, continuity artifact, receipt, memory record, outcome reference, dashboard packet, latest snapshot, or state trace is persisted, that persistence must be explicit, classified, traceable, and governed.

All mutation must pass:

State Authority  
→ Watcher  
→ Canon  
→ Request Governor  
→ Execution Gate  
→ Cross-Core Integrity  

## MEMORY LAW

Memory is advisory.

Memory may preserve context.

Memory may not decide.

Memory may not override governance.

Memory persistence is classified mutation.

## RECEIPT LAW

A receipt is a persisted artifact.

A receipt may document continuity posture.

A receipt may not authorize execution.

A receipt may not mutate workflow truth.

A receipt may not become continuity authority.

## OUTCOME LAW

Outcome references are advisory only.

Outcome persistence is classified mutation.

Outcome references cannot alter workflow, routing, decision, or execution authority.

## EXECUTION LAW

execution_allowed = false by default.

Continuity cannot trigger execution.

Continuity cannot authorize downstream action.

## SUMMARY

Child-core continuity preserves bounded context from lawful upstream flow.

It cannot mutate, execute, retrieve providers, or create authority.
