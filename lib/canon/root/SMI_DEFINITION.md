# SMI DEFINITION

## PURPOSE

SMI is the System Memory and Integrity layer for AI_GO.

It preserves governed continuity context so the system can understand its current posture without treating memory as authority.

SMI does not execute.

SMI does not mutate workflow truth.

SMI does not override governance.

## ROLE

SMI may preserve and surface:

- system posture
- continuity context
- historical records
- run metadata
- memory references
- advisory system summaries
- child-core continuity context

## CHILD-CORE FLOW RULE

Any child-core continuity reference must preserve the lawful upstream flow:

source or provider signal  
→ RESEARCH_CORE  
→ engine curation  
→ adapter  
→ child core  

Child cores may not fetch providers directly.

Child cores may not receive raw provider material as authority.

Child-core continuity is bounded context only.

## AUTHORITY BOUNDARY

SMI may:

- provide context
- expose continuity
- support operator understanding
- support System Brain advisory posture
- support governance review

SMI may not:

- execute
- mutate workflow
- mutate project truth
- override Request Governor
- override Watcher
- override Execution Gate
- override State Authority
- override Cross-Core Integrity
- convert memory into command authority

## MEMORY LAW

Memory is advisory.

Memory may inform review, but memory may not decide.

Memory may preserve context, but it may not override governance.

## MUTATION LAW

Persistence = Mutation.

Any persisted SMI artifact must be classified and traceable.

Allowed SMI-related persistence must remain advisory unless explicitly governed by the full mutation chain.

## EXECUTION LAW

execution_allowed = false

SMI cannot authorize execution.

Execution must pass:

State Authority  
→ Watcher  
→ Canon  
→ Request Governor  
→ Execution Gate  
→ Cross-Core Integrity  

## SUMMARY

SMI preserves continuity without becoming command authority.

It is a memory and integrity layer, not an execution layer.
