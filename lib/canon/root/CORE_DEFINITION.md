# CORE Definition

## Purpose

This document defines the canonical meaning of CORE within AI_GO.

It exists to preserve the role of CORE as the runtime governance authority of the system.

---

## What CORE Is

CORE is the bounded runtime governance surface of AI_GO.

It is responsible for:

- lawful runtime startup after boot
- routing execution across declared surfaces
- maintaining runtime status visibility
- governing continuity and monitoring coordination at runtime level
- preserving implementation boundaries between runtime authorities

CORE is not a research layer, PM layer, engine layer, or child execution layer.

---

## CORE Responsibilities

CORE includes responsibility for:

- runtime
- continuity integration surfaces
- monitoring integration surfaces
- shared infrastructure needed by root runtime authorities

CORE governs how the system runs.

It does not define all meanings inside the system.

---

## CORE Boundary

CORE must not silently assume:

- research ownership
- PM strategic ownership
- child-core execution ownership
- archive ownership
- canon authorship

It may coordinate runtime flow, but coordination is not the same as identity replacement.

---

## Relationship to Other Authorities

### Boot
Boot initializes lawful startup before CORE runtime begins.

### SMI
SMI is continuity authority, often implemented beneath CORE surfaces but not reducible to runtime alone.

### Watcher and Sentinel
Monitoring may operate through CORE implementation surfaces, but monitoring remains its own authority.

### Research
Research may enter runtime pathways, but CORE does not become research authority.

### PM
CORE may route PM surfaces, but CORE does not become planning authority.

### Engines
CORE may host runtime access to engines, but CORE does not become refinement authority.

### Child Cores
CORE may coordinate system-level routing toward child cores, but it does not execute child-core domain work.

---

## Summary

`CORE_DEFINITION.md` preserves the meaning of CORE as the bounded runtime governance authority of AI_GO.

CORE governs runtime flow and structural execution boundaries without replacing the other root authorities of the system.
