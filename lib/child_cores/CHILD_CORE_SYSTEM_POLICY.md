# CHILD CORE SYSTEM POLICY (NORTHSTAR ALIGNED)

## PURPOSE

Defines the role and limits of all child cores inside AI_GO.

Child cores are:

- bounded processors
- domain-specific logic units
- non-authoritative system components

They are NOT:

- sources of truth
- execution authorities
- mutation authorities

---

## SYSTEM FLOW LAW (ENFORCED)

All input MUST follow:

External Signal  
→ RESEARCH_CORE  
→ Engine Layer  
→ Adapter  
→ Child Core  

No exceptions.

---

## CORE PRINCIPLE

Child cores NEVER see raw reality.

They ONLY see:

→ curated  
→ filtered  
→ schema-controlled context

---

## AUTHORITY LAW

Child cores:

- do NOT hold authority
- do NOT define truth
- do NOT approve execution

They operate under:

→ Request Governor  
→ Watcher System  
→ State Authority  
→ Canon Runtime  

---

## MUTATION LAW (CRITICAL)

Persistence = Mutation

ALL mutation must pass:

1. State Authority
2. Watcher Validation
3. Canon Validation
4. Request Governor
5. Execution Gate
6. Cross-Core Enforcement

Child cores cannot bypass ANY layer.

---

## EXECUTION LAW

execution_allowed = false (default system state)

Child cores cannot:

- execute actions
- trigger system changes
- perform real-world effects

---

## OUTPUT LAW

Child cores may ONLY emit:

- bounded artifacts
- advisory outputs
- governed results

All outputs must:

- reflect curated input
- include authority boundaries
- be classifiable
