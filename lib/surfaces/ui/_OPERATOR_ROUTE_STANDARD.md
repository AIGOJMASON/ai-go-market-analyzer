# UI OPERATOR ROUTE STANDARD

## PURPOSE

Defines lawful UI operator-route behavior for AI_GO surfaces.

UI operator routes provide visibility and governed request entry only.

UI operator routes do not execute.

UI operator routes do not mutate by default.

UI operator routes do not create authority.

## CHILD-CORE FLOW LAW

Any child-core data exposed through a UI operator route must preserve:

source or provider signal  
→ RESEARCH_CORE  
→ engine curation  
→ adapter  
→ child-core ingress  
→ bounded child-core processing  
→ UI operator-route visibility  

Provider retrieval is reserved for RESEARCH_CORE only.

Child-core UI data is bounded, curated, adapter-shaped, and advisory unless a governed mutation has been explicitly approved.

## UI OPERATOR ROUTE MAY

- display bounded child-core posture
- display lineage metadata
- display advisory output
- display execution_allowed = false
- display mutation status
- accept governed request packets
- support operator understanding

## UI OPERATOR ROUTE MAY NOT

- execute actions by visibility alone
- mutate workflow by display alone
- mutate project truth by display alone
- write memory outside governance
- write outcome feedback outside governance
- create child-core authority
- create recommendation authority
- bypass governance

## MUTATION LAW

Persistence = Mutation.

If UI operator-route evidence, dashboard packet, receipt, memory record, outcome reference, routing trace, or latest snapshot is persisted, it must be explicit, classified, traceable, and governed.

All mutation must pass:

State Authority  
→ Watcher  
→ Canon  
→ Request Governor  
→ Execution Gate  
→ Cross-Core Integrity  

## RECEIPT LAW

A receipt is a persisted artifact.

A receipt may document governed UI operator-route activity.

A receipt may not authorize execution.

A receipt may not mutate workflow truth by itself.

## DASHBOARD LAW

Dashboard visibility is advisory.

A dashboard packet may display system state.

A dashboard packet may not authorize execution.

A dashboard packet may not mutate workflow truth by itself.

## EXECUTION LAW

execution_allowed = false by default.

UI operator-route visibility cannot trigger execution.

A governed request must pass the full governance chain before any mutation or execution may occur.

## SUMMARY

UI operator routes expose bounded visibility and governed request entry only.

They cannot mutate, execute, retrieve providers, or create child-core authority.
