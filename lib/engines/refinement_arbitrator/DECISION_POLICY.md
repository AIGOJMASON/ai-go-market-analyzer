# REFINEMENT ARBITRATOR DECISION POLICY

## Purpose

Defines how decisions are formed within the refinement arbitrator.

---

## Source Law

All decisions must be derived from:

source or provider signal  
→ RESEARCH_CORE  
→ engine refinement  
→ adapter  
→ arbitrator  

No direct provider or child-core sourcing allowed.

---

## Decision Nature

Decisions are:

- advisory
- structured
- non-executing

They define:

- possible actions
- risk boundaries
- routing direction

---

## Memory and Persistence Clarification

Any reference to:

- memory
- persistence
- stored decision

is classified as **mutation**

---

## Mutation Rules

All mutation must:

- be explicitly declared
- be classified
- be traceable

---

## Mutation Enforcement

All mutation must pass:

State Authority  
→ Watcher  
→ Canon  
→ Request Governor  
→ Execution Gate  
→ Cross-core enforcement  

---

## Prohibited Behavior

- autonomous decision execution
- implicit mutation through decision storage
- bypass of PM or routing layers

---

## Summary

The arbitrator decides.

It does not execute.

All mutation is governed.
