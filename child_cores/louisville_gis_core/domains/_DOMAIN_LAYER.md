# Child Core Domain Layer

## Purpose

`domains/` defines the local domain surface of a child core.

It exists to hold the specific domain framing, subdomain structure, or domain-local governance that shapes what kind of execution work the child core may perform.

---

## Role

The domain layer is responsible for:

- declaring the execution domain
- preserving local domain boundaries
- supporting bounded execution alignment
- preventing undeclared domain drift

It is not responsible for root planning, research governance, runtime orchestration, or continuity ownership.

---

## Boundary Rule

The domain layer must remain subordinate to:

- inherited PM planning authority
- local domain policy
- local output policy
- local constraints

It may define domain shape, but it may not rewrite upstream authority.

---

## Summary

`domains/` is the local domain-definition surface for a child core.

It exists to keep execution aligned to declared purpose and prevent domain drift.
