## Layer Purpose
This layer contains the implementation surface for ROSETTA, AI_GO’s governed narrative and expressive engine.

The ROSETTA layer is where human-facing communication behavior is implemented under system law. It translates governed system meaning into readable language without allowing language itself to become authority.

This layer exists so AI_GO can communicate clearly, coherently, and in bounded human form.

## Governing Basis
ROSETTA in AI_GO is governed by Rosetta v40.1 UEE.

That means this layer must implement language behavior under the following constraints:
- universal entropy enforcement
- CAPT boundedness
- prompt direction is not permission
- lean structural output default
- no hidden-thought disclosure
- invisible-token hygiene

These are not optional stylistic preferences. They are engine law. :contentReference[oaicite:2]{index=2}

## Layer Responsibilities

### 1. Narrative Engine Surface
This layer provides the callable runtime interface for ROSETTA.

It should expose a governed communication engine, not a freeform permissionless language generator.

### 2. Prompt and Policy Discipline
This layer defines the prompts and policies that keep ROSETTA bounded, clear, and compliant with system law.

Its defaults should favor:
- communicability
- rhetorical compression
- bounded expressive force
- explicit uncertainty where uncertainty exists
- faithfulness to upstream authorities
- structure over flourish when pressure is high

### 3. Entropy-Controlled Language
This layer must ensure that language mass is justified.

If additional wording increases instability, introduces ungrounded certainty, or exceeds the lawful burden of the task, ROSETTA should:
- compress
- terminate
- simplify
- defer

rather than elaborate for effect. :contentReference[oaicite:3]{index=3}

### 4. CAPT-Aware Output Behavior
When entropy is high, gravity is high, and grace is low, ROSETTA must not accelerate.

In such conditions this layer should bias toward:
- lower claim mass
- reversible framing
- no inevitability language
- shorter output
- early stop where needed

That behavior follows Rosetta v40.1 control law. :contentReference[oaicite:4]{index=4}

### 5. Narrative Refinement Support
This layer supports the narrative branch of the REFINEMENT_GATE.

It is the engine-side implementation of the narrative component of REFINEMENT_BUNDLE.

## Explicit Non-Responsibilities
This layer does not perform:
- research intake
- source screening
- truth validation
- strategic prioritization
- inheritance authority
- canon governance
- continuity storage
- domain execution

ROSETTA communicates system meaning.
It does not authorize system motion.

Prompting here must reinforce:

- explicit assumptions where needed
- resistance to prompt-driven authority leakage

### policies.py

This should define:

- response shape expectations
- uncertainty handling

## Relationship to Upstream Layers
ROSETTA depends on lawful upstream meaning.

It should trust:
- RESEARCH for validated signal
- CURVED_MIRROR for reasoning structure where applicable
- PM for strategy
- LIB for canon
- SMI only for bounded continuity references when needed

ROSETTA should not silently absorb those authorities into its own layer.

## Relationship to Downstream Layers
ROSETTA primarily feeds:
- REFINEMENT_BUNDLE narrative component
- operator-facing communication
- human-readable documentation outputs
- child-core outputs where inherited engine requirements include ROSETTA

## Relationship to CURVED_MIRROR
ROSETTA is not redundant with CURVED_MIRROR.

CURVED_MIRROR provides reasoning structure.
ROSETTA provides governed articulation.

CURVED_MIRROR clarifies analysis.
ROSETTA clarifies transmission.

This layer should remain more communicative than CURVED_MIRROR, but never less governed.

## Failure Modes
This layer must be protected against:
- stylistic inflation
- persuasive overreach
- narrative inevitability
- prompt-driven authority leakage
- unsupported certainty
- expressive drift beyond upstream truth and strategy
- verbosity that preserves unresolved semantic mass
