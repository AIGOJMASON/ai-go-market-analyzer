# AI_GO

AI_GO is a governed AI operating system built as a bounded authority architecture.

The system separates research, planning, refinement, execution, monitoring, and archival preservation into explicit layers.

---

## Core Principles

- explicit authority separation
- governed information flow
- packetized handoff between system layers
- canon and runtime separation
- preserved provenance across artifacts

---

## Major Root Layers

- boot
- core
- RESEARCH_CORE
- PM_CORE
- engines
- child_cores
- lib
- state
- packets
- telemetry
- interfaces
- tests
- tools

---

## System Philosophy

AI_GO treats architecture as governance.

Each layer performs a specific role, and no layer silently replaces another.
